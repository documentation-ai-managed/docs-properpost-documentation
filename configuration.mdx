## Overview

Properpost provides flexible configuration options to match your team's workflow. You customize themes for branding, manage users and roles for security, connect integrations for extended functionality, and set up backups for data safety. Access these settings from the Admin Dashboard at `https://app.properpost.com/admin/settings`.

## Theme and Branding Setup

Tailor Properpost's appearance to your brand. Update colors, logos, and fonts via the Theme Editor.

<Steps>
  <Step title="Access Theme Settings" icon="palette">
    Navigate to Admin > Themes.
  </Step>
  <Step title="Upload Assets" icon="upload">
    Add your logo (PNG, `<200KB`) and favicon.
  </Step>
  <Step title="Set Colors" icon="droplet">
    Enter your primary color, such as `#040404`.
  </Step>
  <Step title="Preview and Save" icon="eye">
    Preview changes and publish.
  </Step>
</Steps>

<CodeGroup tabs="JSON,Environment">
```json
{
  "primaryColor": "#040404",
  "logoUrl": "https://yourdomain.com/logo.png",
  "fontFamily": "Inter, sans-serif"
}
```

```bash
PROPERPOST_PRIMARY_COLOR=#040404
PROPERPOST_LOGO_URL=https://yourdomain.com/logo.png
```
</CodeGroup>

<Callout kind="tip">
  Test themes in preview mode before publishing to avoid disrupting users.
</Callout>

## User and Role Management

Control access with predefined roles. Assign permissions to teams for secure collaboration.

| Role       | View Posts | Edit Posts | Manage Users | Admin Access |
|------------|------------|------------|--------------|--------------|
| Viewer     | ✅         | ❌         | ❌           | ❌           |
| Editor     | ✅         | ✅         | ❌           | ❌           |
| Admin      | ✅         | ✅         | ✅           | ✅           |

<Expandable title="Custom Role Creation" default-open="false">

Create custom roles via API or dashboard:

```javascript
await properpost.roles.create({
  name: "Content Lead",
  permissions: ["posts:read", "posts:edit", "analytics:view"]
});
```

</Expandable>

## Backup and Export Settings

Protect your data with scheduled backups and exports.

Enable daily backups in Admin > Backups. Exports include posts, users, and settings in JSON or CSV.

```json
{
  "backupSchedule": "0 2 * * *",
  "retentionDays": 30,
  "exportFormat": "json"
}
```

<Callout kind="alert">
  Verify backups monthly to ensure data integrity.
</Callout>
