## Recent Releases

<Update label="2024-10-15" description="v2.1.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration for docs editing with multiple users.
- Introduced custom themes with your brand color `{ #040404 }` support.

## Improvements

- Optimized search indexing for `{<100ms}` query times.
- Enhanced export options including PDF and Markdown bundles.

## Bug Fixes

- Fixed sidebar navigation collapse on mobile devices.
- Resolved API rate limiting issues during bulk imports.

</Update>

<Update label="2024-09-20" description="v2.0.0" tags={["breaking", "feature"]}>

## New Features

- Launched Properpost API v2 with endpoints for automated doc posting.
- Integrated version control with GitHub sync.

## Breaking Changes

- Updated authentication to use `{API_KEY}` headers instead of query params.
- Removed legacy Markdown parser; migrate to new engine.

## Bug Fixes

- Patched security vulnerability in file uploads (CVE-2024-1234).
- Corrected table rendering in preview mode.

</Update>

## Upgrading Properpost

Follow these steps to upgrade to the latest version. Always back up your data first.

<Steps>
  <Step title="Backup" icon="database">
    Export your docs via the admin panel.
  </Step>
  <Step title="Update" icon="download">
    Use your package manager to install the new version.
    
    <CodeGroup tabs="npm,yarn">
      ```bash
      npm update properpost@latest
      ```
      ```bash
      yarn upgrade properpost@latest
      ```
    </CodeGroup>
  </Step>
  <Step title="Migrate" icon="settings">
    Run the migration script if you encounter breaking changes.
    
    ```bash
    npx properpost migrate
    ```
  </Step>
  <Step title="Restart" icon="refresh-cw">
    Restart your server and verify the update.
  </Step>
</Steps>

<Tabs>
  <Tab title="Docker" icon="docker">
    Pull the latest image:
    ```bash
    docker pull properpost/properpost:latest
    docker-compose up -d
    ```
  </Tab>
  <Tab title="Self-Hosted" icon="server">
    Update via Helm or direct binary replacement.
  </Tab>
</Tabs>

<Callout kind="tip">
  Test upgrades in a staging environment first to avoid disruptions.
</Callout>
